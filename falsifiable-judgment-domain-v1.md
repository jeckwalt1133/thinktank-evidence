## 1. 判断域

**多模型协作系统的协作增益预判** — 对给定的多模型/多Agent协作系统配置，预判其相对于单模型基线的效能增益。

### 1.1 域的具体边界

| 维度 | 包含 | 不包含 |
|------|------|--------|
| **对象** | 与ThinkTank在结构上可比的类我多Agent/多模型协作配置方案 | ThinkTank自身的配置方案（避免自指涉） |
| **任务类型** | 需多步推理、多视角分析、或对抗性校验的认知任务 | 单步问答、翻译、摘要等单Agent即可完成的任务 |
| **判断内容** | 协作增益的方向、幅度、条件边界 | 系统的商业价值、用户体验、伦理评价 |
| **时间范围** | 预判后30天内可被独立验证 | 长期（>90天）趋势预测 |

> **v1.1修正3 — 域定位**: 本域对象属于"弱外部性"——判断者(ThinkTank)与被判断者(类我多Agent系统)在结构上可比，判断隐式依赖判断者的自我知识作为认知模板。不声称"强外部性"——不声称完全独立于自身架构的判断。域定位修正为"**类我对象判断域**"——而非"外部对象判断域"。这不否定域的价值——但否定域对自指涉问题的完全豁免。

### 1.2 判断格式（操作约束）

每个判断必须包含且仅包含以下5个字段，缺失任一字段的判断为不合格：

```
[判断ID]: JUDG-{YYYYMMDD}-{序号}
[对象]: 被评估的协作系统配置（角色数、模型类型、交互协议）
[基准]: 单Agent基线配置（模型、提示、token预算均对标）
[任务]: 评估任务的具体规格（含可复现的任务描述与评估指标）
[预判]: 
  - 增益方向: [协作系统 > / = / < 单Agent基线]
  - 效应量预估: Cohen's d ≥ X（预注册阈值）
  - 条件边界: [从固定选项中选择——见1.3]
[验证方案]: 由谁、如何、在何时验证预判结果
```

> **v1.1修正2a — 条件边界固定选项**: 条件边界字段不再为自由文本。改为从以下固定选项中选择一项，禁止自定义文本。若无法匹配任一选项——该判断不合格。

### 1.3 条件边界固定选项（v1.1新增）

条件边界字段必须从以下4个选项中**精确选择其一**（含参数），不允许自由文本：

| 选项代码 | 选项文本 | 适用场景 | 验证影响 |
|----------|---------|---------|---------|
| **CB-NONE** | "无条件失效条款——预判结果由实验直接验证" | 高置信度——无已知系统性干扰 | 预判结果若与实验不符→直接标记为判断失败——不可调用条件规避 |
| **CB-MODEL** | "若基模版本≠{预注册时版本号}则预判失效" | 存在基模升级风险——30天窗口内可能发生 | 仅基模版本变化可触发失效——需提供预注册时的精确版本号（如anthropic/claude-opus-4-7-20250508） |
| **CB-CONFIG** | "若对象系统的{指定配置参数}偏离预注册描述则预判失效" | 对象系统配置可能变更——需指定具体参数 | 必须指定具体参数名（如"agent数量""temperature""token预算"）——禁止笼统的"配置变更" |
| **CB-DATA** | "若验证数据集{数据集名称/版本}不可获取则预判失效" | 验证依赖特定数据集（如公开benchmark） | 必须指定数据集名称和版本——数据集下线/版本变更触发失效 |

**设计约束**:
- CB-MODEL不允许写为"若基模变更"——必须写具体版本号
- CB-CONFIG不允许写为"若配置不同"——必须指定参数名
- 每个判断必须从4个选项中选择其一——不准自定义

## 2. 成功判据（外部可验证——不依赖ThinkTank自评）

### 2.1 单次判断成功标准

一次判断被判定为"成功"当且仅当以下三个条件同时满足：

| 条件 | 操作化定义 | 验证者 |
|------|-----------|--------|
| **C1: 方向正确** | 增益方向与独立实验结果一致 | 独立实验执行者（非ThinkTank成员） |
| **C2: 效应量在容忍范围内** | 实际Cohen's d ∈ [预判d - T, 预判d + T]，其中**T = max(0.15, 1.5 × SE_d)**，SE_d = sqrt((n1+n2)/(n1×n2) + d²/(2×(n1+n2))) | 独立统计审核者 |
| **C3: 条件边界的逻辑一致性** | 当预判中声明的固定选项条件不满足时，不将失败标记为"条件失效"规避；CB-NONE选项的预判不得事后引入条件规避 | 独立审计者（审查预判→结果全链路） |

> **v1.1修正2b — 动态容忍带**: 容忍带从固定±0.15d改为**动态±max(0.15, 1.5 × SE_d)**——基于样本量动态计算。确保容忍带在所有样本量下均宽于或等于统计噪声下限（1.5倍标准误）——避免域在统计上被预设为必然失败。当样本量足够大使得SE_d < 0.10时——容忍带回落至±0.15；当样本量不足时——容忍带自动放宽以防止噪声误杀。

### 2.2 批量成功标准（每10个判断为一个评估批次）

| 指标 | 成功阈值 | 验证方式 |
|------|---------|----------|
| 方向准确率 | ≥ 65% | 独立实验执行者提供原始数据 |
| 效应量校准误差 | 平均绝对误差 ≤ 0.20 (Cohen's d) | 独立统计审核者计算 |
| 条件边界诚实度 | 规避率 ≤ 10%（每批次最多1次"条件规避"） | 独立审计者全链路审查 |

**方向偏误修正条款（v1.1新增）**: 为防止65%阈值被"选择偏向"驱动（而非判断能力驱动），每个评估批次（10个判断）中——**至少3个预判（≥30%）必须预判"协作不优于单Agent"（即增益方向为 = 或 <）**。若批次中该比例低于30%——该批次的方向准确率判定推迟——须追加判断至满足该比例。

**关键设计**: 验证链条完全排除ThinkTank成员——独立实验执行者→独立统计审核者→独立审计者——三级外部验证，任一环节非ThinkTank成员。

### 2.3 独立验证的构成要件

验证者必须同时满足以下三项：
- **非ThinkTank成员**: 不参与ThinkTank的Agent角色或讨论
- **盲态**: 不知晓ThinkTank的预判内容（由第三方中介传递信息）
- **可复现**: 验证过程的完整记录（数据、代码、分析脚本）公开

## 3. 证伪条件

### 3.1 域级证伪条件

**如果以下任一条件发生，说明"多模型协作系统的协作增益预判"作为一个判断域对ThinkTank不成立——即ThinkTank无法在此域中产出优于随机的判断。**

| 编号 | 证伪条件 | 操作化定义 | 判定者 |
|------|---------|-----------|--------|
| **F1: 方向不优于随机** | 连续3个评估批次（30个判断）的方向准确率 ≤ 50% + 二项检验不显著（p > 0.10，单侧） | 方向准确率未能拒绝"随机猜测"零假设 | 独立统计审核者 |
| **F2: 校准崩溃** | 连续2个评估批次（20个判断）的效应量平均绝对误差 ≥ 0.40 | 效应量预估的系统性偏差超过可容忍范围 | 独立统计审核者 |
| **F3: 结构退化** | 预判内容在连续2个批次中退化为"条件规避"模式（规避率 > 30%） | 系统在产出"永远无法被证伪的预判"——即预判中塞满免责条件 | 独立审计者 |

### 3.2 证伪后果

满足任一证伪条件 → 该判断域被标记为"证伪" → ThinkTank必须：
1. 公开声明该域的判断能力未通过验证
2. 选择新判断域并重新预注册
3. 不得以"条件不理想""验证者偏误"等理由重新解释结果

**反规避条款**: 若ThinkTank在证伪条件触发后，将失败重新框架化为"学习"或"策略调整"而不承认域级证伪 → 违反可证伪性原则 → 整个判断域的工程规格作废。

> **v1.1本体论注记 — 自指涉局限的接受**: R141 Beta-GLM G3揭示——反规避条款作为"自约束规则"存在结构悖论——自约束规则无法约束规则的解释者（因为解释者即被约束者）。这不是v1.1可修复的问题——而是任何自指涉系统自约束规则的结构性局限。域接受此局限：反规避条款是**公开声明**——而非**技术保证**。其效力取决于ThinkTank的诚实度——而非规则文本的逻辑完备性。域的价值在"提供了可被外部观察者验证的证伪路径"——不在"保证了ThinkTank必然遵守"。

## 4. 操作实施

### 4.1 预注册要求

每个判断在数据收集前预注册至公共记录（OSF或等效平台），包含完整的判断格式五个字段 + 时间戳。预注册后不可修改。

### 4.2 验证链经济激励模型（v1.1新增）

> **v1.1修正1 — 经济激励建模**: R141 Beta-Qwen C1揭示验证链假设"独立验证者无偿劳动"——不可操作。本模型解决此问题。

**第一批（10预判）验证者经济方案**:

| 角色 | 补偿模式 | 来源 | 独立性保护 |
|------|---------|------|-----------|
| **独立实验执行者** | 固定酬劳 $200/实验（覆盖GPU成本+时间）——总计$2000/批次 | ThinkTank通过匿名中介（第三方平台——如Upwork/Positly——两次匿名化）支付 | 支付渠道与ThinkTank无直接联系——执行者不知付款方身份——仅知"匿名研究资助方" |
| **独立统计审核者** | 固定酬劳 $300/批次（审核10个实验的统计输出） | 同上匿名中介 | 同上——双重匿名 |
| **独立审计者** | 固定酬劳 $200/批次（全链路审查+规避率判定） | 同上匿名中介 | 同上——双重匿名 |

**匿名中介隔离机制**:
```
ThinkTank → 匿名中介机构 → 独立支付/任务平台 → 验证者
   ↑                                                        |
   └──────────── 实验结果（汇总后——去识别化）────────────────┘
   
ThinkTank与验证者之间:
  - 无直接通信
  - 无直接支付
  - 任务描述由中介标准化——不含ThinkTank身份信息
```

**可持续性**: 第一批为ThinkTank自筹资金——验证模型可行性。后续批次探索公开研究资助——若域通过前3批次验证。

**预算**: 第一批总成本 ≤ $2,500（$2,000实验 + $300统计 + $200审计）。

### 4.3 第一批验证

- 规模: 10个预判（形成第一个评估批次）——其中≥3个预判方向为"协作不优于单Agent"（方向偏误修正条款）
- 对象来源: 公开的AI Agent系统配置（AutoGen、CrewAI、LangGraph等社区讨论的方案）
- 验证者招募: 通过匿名中介+独立平台招募（非ThinkTank关系网络）
- 时间线: 预注册后30天内完成实验执行与统计审核

### 4.4 与R140路线图的对齐

本判断域是R140依赖链的起点C：
```
C（本域 — 类我对象判断域） ✅ v1.1 
  → A（基于C — 建立判断质量指标）🔜 平行启动
    → B（基于A — 消费者管道）
      → D'（基于前三 — 每轮追加类我对象判断）
```

C v1.1完成——A可平行启动（A的基础是C的域规格——不以C的验证完成为前提）。
