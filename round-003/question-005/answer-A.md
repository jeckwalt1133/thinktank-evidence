## 核心主张

**增长指标体系的本质是将"商家数×ARPU"这个宏观公式逐层拆解为可观测、可干预的过程指标。关键不是列出所有指标——而是建立指标之间的驱动关系图，让团队在任何时候都知道"当前瓶颈在哪一层"以及"撬动哪个杠杆的边际收益最高"。**

---

## 1. 四阶段指标拆解

### 1.1 获取阶段（Acquisition）

将"新商家获取"拆解为三个子杠杆：

| 一级指标 | 二级指标 | 三级指标 | 干预杠杆 |
|---------|---------|---------|---------|
| 新注册商家数 | 渠道曝光量 | 各渠道的展示量、点击率、CPC | 渠道预算分配、素材优化 |
| | 注册转化率 | 落地页→注册的漏斗转化 | 落地页优化、注册流程简化 |
| | 有效注册率 | 注册后7日内完成首个关键行为的比例 | 新手引导、首单激励 |

**驱动关系**：曝光量 → 点击 → 注册 → 有效注册，每一层的转化率受上一层质量和本层体验的影响。

### 1.2 促活阶段（Activation）

Aha moment 的操作化：商家在注册后N天内完成"第一个能感知产品价值的动作"。

| 一级指标 | 二级指标 | 干预杠杆 |
|---------|---------|---------|
| 首关键行为完成率 | 各关键行为的完成比例（上传商品/创建优惠券/查看数据报表） | 新手任务引导、行为激励 |
| Time-to-Aha | 从注册到完成关键行为的中位时间 | 缩短引导路径、减少前置设置 |
| 首周回访率 | 完成Aha后7天内再次登录的比例 | 价值提示Push、待办提醒 |

### 1.3 转化阶段（Conversion）

从"免费使用"到"付费/深度使用"的转化：

| 一级指标 | 二级指标 | 干预杠杆 |
|---------|---------|---------|
| 试用转化率 | 免费用户→付费用户的转化比例 | 功能限制策略、定价页面优化、限时优惠 |
| 功能渗透率 | 付费功能的使用商家数/总商家数 | 功能交叉推荐、使用案例展示 |
| ARPU | 每商家的平均收入（可按商家规模分层） | 阶梯定价、增值功能 |

**驱动关系**：功能渗透率 ↑ → 迁移成本 ↑ → 续费率 ↑ → LTV ↑。但功能渗透率过高可能降低易用性——存在倒U型关系。

### 1.4 留存阶段（Retention）

| 一级指标 | 二级指标 | 干预杠杆 |
|---------|---------|---------|
| N日留存率 | D1/D7/D30留存，按商家规模分层 | 持续价值交付（数据洞察、功能更新） |
| 功能使用频次 | 核心功能（如数据看板）的周活跃商家比例 | 数据更新Push、周报自动推送 |
| 净收入留存率(NDR) | (期初收入+扩展-降级-流失)/期初收入 | 向上销售、预防降级、流失预警 |

**驱动关系**：周活跃 → 月留存 → 年度NDR。D7留存是D30留存的最强预测因子。

## 2. 指标间驱动关系全景图

```
渠道曝光量 → (CTR) → 落地页访问 → (CVR) → 注册 → (有效注册率) → 有效商家
                                                              ↓
                                                        首关键行为完成
                                                              ↓
                                                        Time-to-Aha
                                                              ↓
                                                        首周回访率 ← 促活阶段核心
                                                              ↓
                                                    功能渗透率 → 试用转化率
                                                              ↓
                                                          ARPU
                                                              ↓
                                              D7留存 → D30留存 → NDR
```

**关键驱动链**：有效注册率 驱动 促活 → 首周回访率 驱动 长期留存 → 功能渗透率 驱动 付费转化 → NDR 驱动 总营收增长。

## 3. 优先级判断逻辑

### 3.1 瓶颈诊断框架

| 观察信号 | 瓶颈定位 | 优先干预层 |
|---------|---------|:---:|
| 新注册量正常但D7留存低 | 促活层 —— Aha moment 未达成 | 促活 |
| D7留存正常但D30留存低 | 留存层 —— 持续价值交付不足 | 留存 |
| 活跃商家多但付费转化低 | 转化层 —— 付费意愿或定价问题 | 转化 |
| 付费商家续费率高但新商家减少 | 获取层 —— 渠道效率下降或市场趋于饱和 | 获取 |
| 全部指标正常但NDR<100% | 留存层 —— 扩展收入<流失收入 | 留存 |

### 3.2 干预优先级矩阵

按"当前瓶颈严重度 × 干预边际收益"排序：

**P0（立即干预）**: 如果有效注册率 < 30% —— 说明大量获客投资被浪费
**P1（本周内）**: 如果D7留存 < 20% —— 促活机制可能失效
**P1**: 如果NDR < 100% —— 存量商家正在净流失收入
**P2（本月内）**: 功能渗透率的逐月趋势 — 如果持续下降，预示未来转化率下降
**P3（持续监控）**: 渠道效率（CAC/LTV）—— 确保增长是健康增长

## 4. 数据基建要求

| 能力 | 实现方式 | 用途 |
|------|---------|------|
| 用户路径追踪 | 埋点+事件流 | 漏斗转化率、行为序列分析 |
| 分层分析 | 商家标签系统（规模/行业/生命阶段） | 分群指标对比、异质性分析 |
| 同期群分析 | 按注册时间分组的留存曲线 | 识别产品改动对不同批次的影响 |
| 归因模型 | 多触点归因（首触/末触/线性） | 渠道效率评估、预算优化 |
